Chapter I

The
Fractured
Age.

The year is now. Billions of machines walk the Earth: in factories, hospitals, warehouses, and homes. They are powerful. They are capable. They are alone.

An Arduino cannot speak to a Raspberry Pi. An ESP32 stares blankly at a ROS2 node. Every board is an island. Every programmer rewrites the same logic, over and over, in five different dialects: C++, Python, MicroPython, another C++, yet more C++.

The corporations called it "the natural order." The engineers called it hell.

"I didn't want to learn five languages to control one robot. I wanted to tell it what to do, in words a human actually uses." — Soham Mulik, creator of Kinetrix
Chapter II

The Villain:
Fragment.

Fragment has no face. No boardroom. No CEO. It is a system: the quiet, profitable force of incompatibility that the hardware industry chose to maintain rather than fix.

Proprietary SDKs that lock your code to one board forever.
Incompatible protocols. C++ on Arduino, Python on Pi, MicroPython on Pico, a different C++ on ESP32. Four boards, four entirely separate codebases.
No fleet control. You want to update 200 warehouse robots? Good luck. One. USB. At. A. Time.
No cross-board tasks. Your sensor runs on Pi. Your motor runs on Arduino. They cannot talk. That's a problem Kinetrix solves.
Complexity as a barrier. The biggest problem in robotics isn't hardware. It's convincing beginners that "this is just how it works."
Chapter III

The
Engineer.

Soham Mulik. A mad monk with a compiler. He had stared at the Fractured Age long enough. He had rewritten the same blink program in C++, then Python, then MicroPython, then C++ again, then yet more C++.

He didn't build a bridge between languages. Bridges still serve Fragment; you still need both shores. Instead, he burned the old roads and paved a single new one.

The rule was simple: "If a student can't read it out loud and understand it, it has no place in the language." And so turn on pin 13 was born. And the world changed.

Chapter IV

The Hero:
AXIOM.

Axiom is the compiled core of Kinetrix, the engine that takes one program and produces native code for five different boards. You write your logic once, in plain readable syntax. Kinetrix handles the translation.

🗣️
One Syntax
turn on pin 13. Human words. Axiom translates to native C/C++/Python for each board. You write once. It speaks five languages.
⚡
Native Performance
Axiom compiles to native code. No interpreter. No VM overhead. Pure compiled C++ running bare-metal on your microcontroller.
🌐
Fleet Updates
One command pushes new firmware to every robot in your fleet over WiFi. No cables. No manual uploads. OTA built into the language itself.
🧠
Concurrent Tasks
Native task system lets a robot's sensors, motors, and comms run simultaneously, without manual threading or GIL limitations.
# Axiom speaks. Every machine listens.
program {
  enable ota "my_fleet" password "secret"   enable watchdog timeout 5000ms
  shared make number sensor = 0
  task read_world {     sensor = read analog pin 34     feed watchdog     wait 50   }
  task act_on_world {     set pin 9 to sensor     radio send 0 sensor     wait 20   }
  on interrupt pin 2 rising {     send serial "OVERRIDE"   }
  start task read_world   start task act_on_world }
# ./kcc axiom.kx --target esp32 -o out.cpp # ./kcc axiom.kx --target rpi -o out.py # ./kcc axiom.kx --target arduino -o out.ino
Chapter V

The Five
Awakened.

Once scattered across incompatible SDKs, each board now speaks through the same language. Each powerful on its own. More powerful working together.

I 🤖
Arduino
Uno · Nano · Mega

The original soldier. Once chained to digitalWrite(13, HIGH). Now it just hears "turn on pin 13" and obeys.

--target arduino
II 📡
ESP32
Dev Module · ESP-NOW

The wireless one. WiFi, OTA fleet updates, radio mesh, I2C. All available through Kinetrix with a single compile flag.

--target esp32
III 🍓
Raspberry Pi
Pi 3 · Pi 4 · Pi 5

The thinker. Full Linux. Full GPIO. Full power. Kinetrix compiles to Python + RPi.GPIO, giving the Pi a syntax that actually feels human.

--target rpi
IV 🎯
Pico / Pico W
RP2040 · MicroPython

The quick one. Tiny, silent, precise. Kinetrix outputs MicroPython that runs blazing fast on 2MB of flash memory.

--target pico
V 🦾
ROS2
Robot Operating System 2

The general. Production-grade robotics. Kinetrix compiles to full ROS2 C++ nodes, for the robots that move the real world.

--target ros2
Chapter VI

The Arsenal:
Every Weapon Axiom Carries.

Twelve powers. One syntax. All of them free.

// WEAPON 01
GPIO Control
Digital, analog, PWM. All in plain English. No register maps. No bit-shifting.
turn on pin 13
set pin 9 to 128  # PWM
read analog pin 34
// WEAPON 02
Concurrent Tasks
Run sensor, motor, and comms loops simultaneously with shared memory and no thread hell.
task sensors { ... }
task motors { ... }
start task sensors
// WEAPON 03
OTA Fleet Updates
Push firmware to 200 robots over WiFi with a single shell command. Fragment's nightmare.
enable ota "fleet" password "key"
# From laptop:
$ ./kcc_push.sh robot.kx --target esp32
// WEAPON 04
I2C / SPI / Serial
Talk to sensors, displays, and modules over every hardware protocol without deciphering datasheets.
open i2c
read i2c device 0x68 register 0x41
send serial "Hello"
// WEAPON 05
Safety Systems
Watchdog timers, try/on-error blocks, and assert guards. Robots that fail safely are robots worth trusting.
enable watchdog timeout 5000ms
try { ... } on error { ... }
assert speed < 255
// WEAPON 06
Interrupts & Timers
Hardware interrupts on rising/falling edges. Periodic timers for watchdog, polling, and real-time responses.
on interrupt pin 2 rising { ... }
on timer every 5000ms { ... }
// WEAPON 07
ESP-NOW Wireless
Peer-to-peer radio between ESP32 robots. No router. No internet. Direct mesh communication.
radio send 0 sensor_value
if radio available {
  make number msg = radio read
}
// WEAPON 08
Inverse Kinematics
Native floating-point math and IK support. Arm trajectory math belongs in the language, not a library.
make number x = cos(angle) * 100.0
make number y = sin(angle) * 100.0
set pin 9 to x
// WEAPON 09
Structs & Arrays
Typed data structures for sensor payloads. Circular buffers for signal history and smoothing.
define type SensorData { number temp }
make buffer history size 5
push val into history
// WEAPON 10
Functions
Define, reuse, return. Strongly-typed parameters. Recursion works. The building blocks of clean robot logic.
define function clamp(
  val number, lo number, hi number
) returns number { return val }
// WEAPON 11
Package Manager (kpm)
kpm.py and kpm.js let you install and share Kinetrix libraries. The ecosystem is already growing.
$ kpm install servo-driver
$ kpm install pid-controller
$ kpm publish my-sensor-lib

VS Code Support
Full editor setup with syntax highlighting and compile shortcuts. Robots deserve a good IDE experience.

Chapter VII
v3.1.0 — Just Shipped

Library
Wrappers.

V3.1 brings native syntax for the components every robotics project actually uses. Servos, sensors, displays, LEDs. The compiler handles the includes, the boilerplate, and the target-specific library differences automatically. You just describe what you want.

Compiler support live  ·  Hardware testing in progress
⚙️
Servo Motors
auto-includes Servo.h · RPi.GPIO · MicroPython servo
Attach, move, detach. The compiler picks the right servo library for each target automatically.
attach servo pin 9
move servo to 90   # degrees
move servo to 0    # full left
detach servo pin 9
🌡️
DHT11 / Temperature
auto-includes DHT.h · adafruit_dht · dht module
One-line sensor attachment. Read temperature and humidity without touching a datasheet.
attach dht11 pin 4
make float temp = read temperature
make float hum  = read humidity
print temp
📏
Ultrasonic Distance
HC-SR04 · trigger/echo pattern · pulse timing
Distance in centimetres, one expression. No manual pulse timing, no microsecond math.
make float dist = read distance
  trigger 12 echo 11
if dist < 20 {
  turn off pin 9 # stop motor
}
💡
NeoPixel LEDs
auto-includes Adafruit_NeoPixel · rpi_ws281x · neopixel
Addressable RGB strips in three lines. Set individual pixels, show the frame, done.
attach strip pin 6 count 30
set pixel 0 to 255 0 0 # red
set pixel 1 to 0 0 255 # blue
show pixels
🖥️
LCD Displays
16x2 · 20x4 · I2C LCD · LiquidCrystal
Print to an LCD without LiquidCrystal boilerplate. Specify columns, rows, and write.
attach lcd columns 16 rows 2
lcd print "KINETRIX v3.1" line 0
lcd print "Hardware ready" line 1
📦
Standard Libraries
drive.kx · sonar.kx · music.kx · math.kx
Four ready-to-use Kinetrix libraries written in Kinetrix itself. Drop them in and call their functions.
# drive.kx
drive_forward(200)    # pins 3,5
# sonar.kx
get_distance_from(7)
# music.kx
play_alert()
# math.kx
map_to_percent(sensor)
